# Core Concepts

This guide explains the three ideas at the center of how trusty-mpm works:
agents, delegation authority, and instructions. Understanding these will help
you customize your setup, add your own agents, and know exactly what trusty-mpm
controls versus what is yours.

## Agents

An agent is a Claude Code sub-instance with a specific role and set of
constraints. Rather than one Claude Code session trying to do everything,
trusty-mpm lets an orchestrating session delegate tasks to purpose-built
specialists: an engineer agent for code changes, a research agent for
investigation, a QA agent for testing, and so on.

### Built-in Agents

trusty-mpm ships with a set of built-in agents that cover common development
workflows. When you run `trusty-mpm install`, these are deployed to
`~/.claude/agents/` in ready-to-use form. You do not need to configure them —
they work out of the box.

Each built-in agent is composed from a hierarchy of shared foundation files.
The engineer agent, for example, inherits a base engineer layer (code quality
standards, testing conventions, linting rules) which in turn inherits a base
agent layer (memory routing, git workflow, output format, handoff protocol). You
see only the composed result; the hierarchy is trusty-mpm's internal concern.

### Adding Your Own Agents

You can write your own agent definitions and place them in
`~/.trusty-mpm/framework/agents/`. An agent file is a Markdown document with a
small YAML header:

```yaml
---
name: payments-engineer
description: Handles all work in the payments module. Strict scope: never modifies files outside src/payments/.
model: sonnet
extends: base-engineer
---

# Payments Engineer

Only work within `src/payments/`. For any request that would require
changes outside this directory, escalate to the orchestrating session.
```

The `extends: base-engineer` line tells trusty-mpm to compose your agent on top
of the full engineer foundation. You get all the standard engineer behaviors
automatically — your file only needs to describe what is different or additional.

You can also write agents that do not extend any base type, if you want full
control over everything the agent knows.

### Your Agents Are Always Yours

trusty-mpm tracks which agent files it deployed and records a checksum for each
one. If you edit a deployed agent file, trusty-mpm detects the change at the
next session start and leaves your version alone. It will warn you that it
skipped the file, but it will never silently overwrite your edits.

Files that trusty-mpm has never deployed are completely untouched — trusty-mpm
only manages files it created.

If you want to return a modified agent to the framework default, delete your
edited version and run `trusty-mpm session start`. trusty-mpm will redeploy the
default composition.

## Delegation Authority

When you start a new session, trusty-mpm scans your installed agents and
automatically tells the orchestrating Claude Code instance which ones are
available and what each one is for. This is called the delegation authority.

You do not configure this manually. trusty-mpm reads the agents that are
actually deployed and generates the routing information fresh each time. If you
add a new agent and start a new session, it appears in the authority
automatically. If you remove an agent, it disappears from routing without any
additional cleanup step.

The practical effect: the orchestrating session always has an accurate picture
of its available team. It knows to send code work to the engineer agent, not to
guess at what agents exist or whether they are installed.

## Instructions

The instructions delivered to each new Claude Code session come from three
sources, merged in a fixed order:

### 1. Framework Rules (INSTRUCTIONS.md)

trusty-mpm's standing instructions for all managed sessions. This covers how
sessions communicate with the daemon, where framework configuration lives, and
what the behavioral guarantees are. It is installed and updated automatically
by trusty-mpm — you do not edit this file.

### 2. Dynamic Routing (Delegation Authority)

The agent inventory described above. Generated fresh at session start.

### 3. Your Project Notes (CLAUDE.md)

A file seeded for you at `~/.trusty-mpm/framework/instructions/CLAUDE.md`
when you first install or when you register a project with
`trusty-mpm project init`. This file is yours entirely. trusty-mpm never
overwrites it after the initial seed, even when you upgrade the framework.

This is where you put anything that should apply to every session in this
project:

- Technology choices and version constraints
- Repository conventions (branch naming, migration rules, etc.)
- Things the agents should always or never do in this codebase
- Team-specific workflow notes

Edit it with any text editor. Changes take effect the next time you start a
session.

### Changing Behavioral Policies

Because `INSTRUCTIONS.md` tells sessions where framework configuration lives,
sessions can update their own behavior by editing config files directly. The
most common example is the token optimizer policy at
`~/.trusty-mpm/framework/hooks/optimizer.toml`, which controls how aggressively
tool outputs are compressed to save context budget. A session can lower or raise
this setting mid-work, and the change takes effect immediately.

All framework file changes are reversible. Running `trusty-mpm install --force`
resets all framework-owned files to their bundled defaults. Your `CLAUDE.md` is
never reset — it survives every reinstall.

## Summary

| Concept | Who manages it | Where it lives | Can you edit it? |
|---------|---------------|----------------|------------------|
| Built-in agents | trusty-mpm | `~/.claude/agents/` | Yes — edits are detected and preserved |
| Your custom agents | You | `~/.trusty-mpm/framework/agents/` | Yes — completely yours |
| Delegation authority | trusty-mpm (generated) | In-memory at session start | No direct editing; add/remove agents to change it |
| Framework instructions (INSTRUCTIONS.md) | trusty-mpm | `~/.trusty-mpm/framework/instructions/` | Not recommended — reset on `install --force` |
| Project notes (CLAUDE.md) | You | `~/.trusty-mpm/framework/instructions/` | Yes — never reset by trusty-mpm |
| Behavioral policies (optimizer.toml, etc.) | Shared | `~/.trusty-mpm/framework/hooks/` | Yes — changes take effect immediately |
